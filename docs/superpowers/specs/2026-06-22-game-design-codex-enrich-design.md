# Game Design Codex enrichment design

## Goal

Expand the refined Obsidian note with practical design patterns and game-specific context while preserving the original research text as a distinct layer.

## Scope

Add one compact, sourced `#### Дополнение — …` block to each of the twelve mechanic profiles. Each block will give a concrete implementation example, a project consequence, and one prototype check. Use official publisher/rulebook sources where available; leave a profile unchanged when a suitable source cannot be verified.

## Boundaries

Work from a temporary copy of the vault note. Do not edit or delete the vault copy until the enriched note, links, Markdown structure, and source attribution have been reviewed. The final transfer replaces the existing note only after explicit user confirmation.

## Verification

Confirm that each addition has a descriptive inline link, remains in the exact enrichment block form, does not introduce wide tables or non-vertical Mermaid, and that the original prose remains intact.
